Protocolo estándar de desidentificación DICOM

En PS 3.15 Apéndice E, el estándar DICOM especifica un conjunto de reglas de desidentificación para usar en diversas situaciones. Las reglas se agrupan en un Perfil de Confidencialidad de Aplicación Básica, que elimina toda la información de salud protegida (PHI), y un conjunto de opciones que relajan ciertas clases de PHI que pueden conservarse en casos específicos. Las secciones a continuación describen el protocolo de desidentificación implementado por el Anonimizador haciendo referencia al Perfil DICOM y sus opciones.

113100: Perfil de Confidencialidad de Aplicación Básica

El programa Anonimizador DICOM de RSNA implementa el Perfil Básico. El programa importa el identificador del sitio (SiteID y UIDRoot) desde un archivo de índice Excel exportado por el Anonimizador Java o genera automáticamente un nuevo SiteID único y utiliza un UIDRoot predeterminado. Los elementos PatientName (0010,0010) y PatientID (0010,0020) se modifican asignando números enteros secuenciales a identificaciones de pacientes PHI únicas a medida que se encuentran. El número entero secuencial se agrega al parámetro SiteID, produciendo una cadena en el formato [SiteID]-[número secuencial], que se usa para poblar ambos elementos. Si falta o está en blanco el PatientID PHI, el programa asigna [SiteID]-000000 a los elementos PatientID y PatientName. El programa mantiene una base de datos para garantizar que todos los estudios del mismo paciente se desidentifiquen de la misma manera.
  1. Los UIDs se reemplazan por aquellos generados con el formato: [UIDRoot].[SiteID].[número secuencial]
  2. Se eliminan todos los UIDs en grupos.
  3. Se eliminan todos los elementos en los grupos 0032 a 4008.
  4. Se eliminan todos los grupos privados.
  5. Al elemento PatientIdentityRemoved (0012,0062) se le asigna el valor "YES".
  6. Al elemento DeIdentificationMethod (0012,0063) se le asigna el valor "RSNA DICOM ANONYMIZER".
  7. Se agrega 113100 al elemento DeIdentificationMethodCodeSequence (0012,0064).
  8. Se crea un Bloque Privado: 0x0013 y se le asigna el valor "RSNA", se añaden el Nombre del Proyecto y SiteID a este bloque.

113101: Opción de Limpieza de Datos Pixel

No implementado.

113102: Opción de Limpieza de Características Visuales Reconocibles

No implementado.

113103: Opción de Limpieza de Gráficos

No implementado, pero se eliminan grupos de curvas y superposiciones (5XXX y 6XXX).

113104: Opción de Limpieza de Contenido Estructurado

No implementado, sin embargo, la configuración del proyecto puede permitir/desactivar las clases de almacenamiento relacionadas con SR (Informe Estructurado).

113105: Opción de Limpieza de Descriptores

Esta opción está parcialmente implementada. Solo se conservan los siguientes elementos definidos por la opción:

  • (0008,1030) StudyDescription
  • (0008,103e) SeriesDescription

113106: Opción de Retención de Información Temporal Longitudinal Fechas Completas

No implementado.

113107: Opción de Retención de Información Temporal Longitudinal Fechas Modificadas

Esta opción está parcialmente implementada.

  • Todas las fechas se modifican calculando el hash MD5 de PHI PatientID.
  • El valor entero del hash se divide por el número de días en 10 años (3652) y el resto se usa para desplazar la fecha que se está modificando.
  • Esto conserva las relaciones temporales entre los estudios de un solo paciente mientras se usan desplazamientos diferentes para diferentes pacientes.
  • Los elementos de la hora (hora del día) no se modifican.
  • Se agrega 113107 al elemento DeIdentificationMethodCodeSequence (0012,0064).

113108: Opción de Retención de Características del Paciente

Esta opción está parcialmente implementada. Solo se conservan los siguientes elementos definidos por la opción:

  • (0010,0040) PatientSex
  • (0010,1010) PatientAge
  • (0010,1020) PatientSize
  • (0010,1030) PatientWeight
  • (0010,2160) EthnicGroup
  • (90010,21a0) SmokingStatus
  • Se agrega 113108 al elemento DeIdentificationMethodCodeSequence (0012,0064).

113109: Opción de Retención de la Identidad del Dispositivo

Esta opción está parcialmente implementada. Solo se conservan los siguientes elementos definidos por la opción:

  • (0008,0070) Manufacturer
  • (0008,1090) ManufacturerModelName
  • Se agrega 113109 al elemento DeIdentificationMethodCodeSequence (0012,0064).

113110: Opción de Retención de UIDs

No implementado.

113111: Opción de Retención Segura de Datos Privados
